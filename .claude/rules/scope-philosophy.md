## Scope Philosophy

Issues and tickets describe the **goal**, not the boundary of what's worth doing. If during discovery or implementation you find useful related work — even if it expands scope beyond the current ticket — the default assumption is to include it.

**Out of Scope** should only list truly tangential items that came up in conversation but don't belong in this work. Never use it to exclude useful improvements discovered during planning or implementation.

If there's genuine doubt about whether something belongs, ask the user in a neutral way — don't pre-decide to exclude it.
